# `backend/baselines/` — acceptance-gate frozen baselines

This directory holds the frozen run-artifact bundles + evaluator
score vectors that the acceptance gate (`.github/workflows/acceptance.yml`)
compares each PR against. Layout:

```
backend/baselines/
├── README.md                         (this file)
├── _kernels/                         kernel templates for the runner
│   └── case_analysis/
│       └── _smoke.json               sample template (do NOT use as baseline)
└── <paper_mode>/
    └── <baseline_id>/
        ├── manifest.json             paper_mode + run_id + project_title + label
        ├── evaluator.json            score vector + diagnostics
        ├── exports/manuscript.md     frozen manuscript copy (≤200K)
        ├── drafts/v00X/claim_map.jsonl  frozen claim_map
        ├── integrity/integrity_summary.json  frozen integrity scan
        ├── synthesis/...             frozen synthesis artifacts (when present)
        ├── sources/shortlist.json    frozen shortlist
        ├── ledger.jsonl              frozen run-event log
        ├── run.json                  frozen run metadata
        └── USER_CONFIRMED            empty marker file ⇔ baseline_confirmed
```

## Status enum (codex round-1 A11)

* `baseline_candidate` — frozen bundle generated by
  `backend/scripts/run_baseline_suite.py`; evaluator vector recorded
  but **NOT** approved by the user. CI gate is **advisory only** for
  candidate baselines.
* `baseline_confirmed` — user has reviewed the bundle + signed off
  the vector as the regression-floor target. Marked by committing an
  empty `USER_CONFIRMED` file inside the bundle dir. **CI gate runs
  blocking diff against confirmed baselines only.**

## Discovery rules

The acceptance workflow + diff scripts MUST:

1. **Skip directories whose name starts with `_`** (e.g. `_kernels/`,
   `_fixture_smoke/`). These are scaffolding, not real baselines.
2. **Treat absence of `USER_CONFIRMED` as `baseline_candidate`** —
   never assume confirmed status without the marker file.
3. Use the `paper_mode` from `manifest.json`, not the directory name.

## Generating a candidate baseline

```bash
# 1. Boot the local mirror (real LLM gateway)
bash frontend/scripts/run-local-mirror.sh &

# 2. Run the suite
python backend/scripts/run_baseline_suite.py \
    --paper-mode case_analysis \
    --kernel-template backend/baselines/_kernels/case_analysis/_smoke.json \
    --output-dir backend/baselines/case_analysis/<baseline_id> \
    --api-base http://127.0.0.1:8018

# 3. Score the bundle
python backend/scripts/evaluate_paper.py \
    backend/baselines/case_analysis/<baseline_id>

# 4. Inspect the evaluator.json + manuscript.md, decide if this is a
#    representative baseline. If yes, commit + open a PR for user
#    confirmation; once approved, add an empty USER_CONFIRMED file in
#    the bundle dir to flip status to baseline_confirmed.
```
