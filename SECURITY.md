# Security Policy

## Supported Versions

Security fixes are handled on the `main` branch and the latest tagged release.

## Reporting A Vulnerability

Please report suspected vulnerabilities through GitHub Security Advisories for this repository when available. If advisories are unavailable, open a minimal private report with the maintainers and avoid posting exploit details in a public issue.

## Secrets And Accounts

Do not commit production credentials, provider tokens, private prompts, generated manuscripts from real users, deployment hostnames, private network addresses, or environment-specific operations notes.

This project does not ship a default production account. Deployers are responsible for creating their first administrator account and rotating any local development credentials before exposing an instance.
