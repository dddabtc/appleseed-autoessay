# Appleseed AutoEssay 说明文档

## 1. 这个系统用来做什么

Appleseed AutoEssay 帮研究者在一个可追溯的工作台里推进论文，而不是让一个聊天窗口一次性写完整篇。

用户提供题目、论文语言、研究领域、作者信息和必要说明。系统随后检索文献，请用户审阅文献选择，读取通过的资料，提出写作方向，起草论文，改进稿件，检查风险，并导出文件。

用户仍然是作者。系统不会把论文提交到任何期刊或平台。它会保留记录，让用户知道检索了什么、确认了什么、上传了什么、写了什么、检查出了什么。

## 2. 为什么不能只靠一次大提示

一次提示可以生成很流畅的学术腔文字，但流畅不等于可靠。文本可能引用弱资料，可能把证据撑不住的判断写得太满，也可能在修改中悄悄改变主张。

Appleseed 把工作拆开：

- 先检索可能相关的资料；
- 再让用户判断哪些资料属于这个题目；
- 再整理成较短的阅读清单；
- 再让用户确认这份阅读清单；
- 再基于这些材料起草；
- 最后在导出前改进和检查稿件。

这比聊天回复慢。好处是每个选择都看得见。如果论文方向出错，用户可以回到较早步骤，只重做需要改变的部分。

## 3. 论文流程

### 创建项目

用户创建一次运行，填写清楚的研究问题、领域、论文模式和论文语言。界面语言和论文语言是两项设置：中文界面可以写英文论文，英文界面也可以写中文论文。

中文题目尤其需要具体说明。年份、地区、模型名称、数据来源、产品编码、资料范围等信息越明确，文献检索越不容易跑偏。

### 审阅提案

系统会先准备一份提案，让用户在深入检索前检查方向。如果某次运行没有提案，而用户后来修改研究设定，文献检索会被标记为需要重新审阅，避免后续内容继续沿用旧方向。

### 文献检索

第一轮检索会产生一批粗候选。候选里可能有噪音，尤其是题目关键词会出现在其它领域时。现在工作台在等待检索审阅时会自动打开“已速览”候选，不会让用户误以为文献页为空。

用户可以通过、拒绝或置顶检索候选。保存这次审阅前，下一步不会使用这些粗候选。

### 整理阅读清单

完成检索审阅后，系统会整理较短的阅读清单。它会排序、去重，再检查资料是否贴合题目，并记录哪些材料被保留或剔除。

OpenAlex 和 Crossref 的记录不一定直接给 PDF。Appleseed 现在会用有限的页面查找方式寻找真正的 PDF 链接。普通下载失败时，还可以用浏览器方式再尝试一次。如果这些安全路径都拿不到全文，系统会保留元数据，并提示用户手动上传。

### 上传缺失 PDF

当某条文献卡片提示需要 PDF 时，用户可以直接点击那张卡片旁边的上传按钮。上传文件会自动关联到这条文献。底部的全局上传按钮仍然保留，适合手动补充其它资料。

用户上传的 PDF 会随该次运行的文献文件保存。重跑文献检索或文献整理时，这些上传文件不会被删除。

### 审阅深读集合

系统读取较短清单之前，用户还要再审阅一次。这一步很重要，因为它决定后续论文可以诚实使用哪些材料作为证据。

### 综合、选方向与起草

系统会概括已通过的资料，提出几个论文方向，并等待用户选择。用户选定方向后才开始起草。

草稿应当受限于已确认资料。没有资料支撑的判断应该显露出来，供用户处理，而不是被悄悄写成很有把握的句子。

### 文风、评审与完整性检查

接近结尾时，稿件会在最终检查前经过几轮聚焦改进。这些改进重点看证据是否够、结论是否写得过强、引用是否有问题、章节是否完整，以及哪些地方需要更小范围修改。

最终评审并不代表论文可以不改就投稿。它的作用是让风险在导出前暴露出来。

### 导出

用户接受最终结果后，Appleseed 会导出正文和引用文件。常见输出包括 Markdown、HTML、DOCX、BibTeX、CSL JSON、文献使用表和自检报告。

## 4. 恢复与运行管理

如果运行被阻塞，工作台现在会自动打开对应步骤的页面并显示原因。例如导出检查出了问题，打开运行时会进入导出页，而不是停在无关页面。实时消息也会刷新提示条，继续或重试按钮应当直接出现，不需要先手动刷新。

删除运行只会删除那一次运行，不会把同一论文项目下的其它运行一起删除。符合工作区策略时，已删除运行可以恢复。如果删除或取消请求之后仍有步骤完成，恢复后的运行会显示提醒，提示用户先查看发生了什么。

## 5. 用户仍然需要检查什么

Appleseed 帮你组织流程，但不替代研究判断。使用导出稿之前，用户应检查：

- 文献是否真的属于这个题目；
- 结论是否没有超过证据；
- 重要判断是否都有支撑；
- 上传的 PDF 是否允许用于这项工作；
- 作者姓名、单位和顺序是否正确；
- 最终文件是否符合目标刊物要求。

最稳妥的用法，是把 Appleseed 当作可追溯的起草和审阅工作台，而不是自动投稿系统。
